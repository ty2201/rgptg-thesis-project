# 算法设计说明

## 1. 核心思想

本项目将原生 SoT 的两阶段流程：

1. 生成骨架；
2. 并行扩展骨架节点；

改造为规则引导的 KG-GoT-SoT 混合流程：

1. 查询语义锚定；
2. 知识图谱检索；
3. 结构化骨架生成；
4. 规则校验与依赖补全；
5. DAG 混合调度；
6. 并行扩展；
7. 全局聚合与一致性修复。

## 2. 数据结构

### ThoughtNode

每个思维节点包含：

- `id`：节点编号；
- `claim`：核心论点；
- `entity_id`：绑定的 KG 实体；
- `relation`：节点主关系；
- `depends_on`：前置依赖；
- `confidence`：置信度；
- `verified`：是否已经被知识或规则验证；
- `evidence`：关联三元组；
- `content`：扩展后的文本。

### KnowledgeTriple

知识图谱三元组：

```text
head --relation--> tail
```

用于提供事实证据、隐式依赖和强约束校验。

## 3. 规则设计

### 置信度规则

当节点置信度低于阈值时，系统将节点标记为低置信度，后续聚合时可以降低其权重。

### KG 强约束规则

如果节点生成的关系与知识图谱中的强约束相冲突，系统会剪枝该节点，避免错误分支继续扩展。

### 隐式依赖规则

如果两个节点在 KG 中存在强关联，且依赖方向不会形成环，系统会自动补充依赖边，使调度器保留必要的串行关系。

## 4. 调度策略

系统把思维节点视为 DAG：

- 所有前置依赖完成的节点进入就绪队列；
- 就绪队列中的节点并行扩展；
- 依赖未完成的节点等待；
- 若检测不到可执行节点，则说明存在循环依赖或缺失依赖。

### KG-guided DAG 构建

为了优化简单 SoT 中“扁平骨架列表”的问题，项目新增 `KGGraphBuilder`，在 LLM 扩展之前直接利用知识图谱证据构造思维图结构。

典型结构为：

```text
n1: 语义锚定节点
  -> n2: KG 证据分支 A
  -> n3: KG 证据分支 B
n2 + n3
  -> n4: 语义聚合节点
```

其中：

- `n1` 负责把用户问题锚定到 KG 核心实体；
- `n2`、`n3` 由不同 KG 三元组生成，可以并行扩展；
- `n4` 显式依赖多个证据分支，用于统一术语、消解分支差异并生成综合结论。

这使依赖结构从“LLM 自由生成”变为“KG 证据驱动生成”，从而降低骨架跑偏和并行分支互相割裂的风险。

新增结构指标：

- `graph_structure_score`：衡量是否形成 root anchor、parallel branches、aggregation node，以及聚合节点是否至少依赖两个证据分支。

## 5. 评测指标

当前原型提供轻量级自动评测：

- `latency_ms`：生成耗时；
- `node_count`：节点数量；
- `parallelizable_ratio`：可并行节点比例；
- `verified_ratio`：已验证节点比例；
- `pruned_count`：剪枝节点数量；
- `dependency_count`：依赖边数量；
- `coverage_score`：查询关键词覆盖度；
- `coherence_score`：基于依赖和段落衔接的粗略连贯性评分。

这些指标适合原型阶段调试。正式论文实验中应进一步接入人工评价或模型评价。

## 6. 对照方法

项目当前提供三种实验方法：

- `direct`：直接生成完整答案，不进行显式骨架规划。
- `sot`：原生 Skeleton of Thought，生成骨架后并行扩展，但不使用 KG 证据、规则剪枝和隐式依赖。
- `rule_guided`：本文方法，使用 KG 检索、规则校验、隐式依赖和 DAG 混合调度。

批量实验会对每条任务同时运行多种方法，输出可用于论文实验表格的指标。

## 7. 后续接真实 LLM 的位置

真实模型只需要实现 `rgptg.llm.LLMClient` 三个方法：

- `plan`
- `expand`
- `aggregate`

其余 KG、规则、调度和评测模块都可以复用。
