# 完成项目提纲

## 一、项目定位

项目名称：规则引导的并行文本生成

研究目标：在 SoT 的并行生成优势基础上，引入知识图谱、Graph of Thoughts、规则校验和动态依赖调度，使系统在保持生成速度的同时提升事实准确性、逻辑连贯性和强逻辑任务处理能力。

## 二、总体技术路线

1. 语义锚定与上下文初始化
   - 从用户查询中抽取关键实体和关系。
   - 将实体链接到知识图谱中的标准节点。
   - 检索 k-hop 子图作为初始知识包。

2. 知识引导的发散生成
   - 使用结构化骨架模板生成候选思维节点。
   - 每个节点记录核心论点、绑定实体、关系类型、依赖节点、置信度。
   - KG 邻居路径作为候选分支建议，减少遗漏关键知识点。

3. 动态逻辑校验与依赖构建
   - 检查节点关系是否违反知识图谱强约束。
   - 根据关系类型自动补充隐式依赖。
   - 对冲突节点进行剪枝、降级或标记为待验证。

4. 串行-并行混合调度
   - 将思维节点构建为有向无环图。
   - 无依赖节点并行生成。
   - 有依赖节点等待前置节点完成后再生成。
   - 保留关键路径优先优化的扩展接口。

5. 语义对齐与结构化聚合
   - 统一术语和实体名称。
   - 将分支内容围绕共同上位概念聚合。
   - 对冲突内容按置信度和 KG 权威度裁决。
   - 输出逻辑连贯的最终文本。

## 三、系统模块划分

1. `kg.py`
   - 负责知识图谱加载、实体链接、邻域检索、关系约束查询。

2. `models.py`
   - 定义 `ThoughtNode`、`KnowledgeTriple`、`GenerationResult` 等核心数据结构。

3. `rules.py`
   - 实现事实约束、依赖补全、置信度降级和剪枝规则。

4. `scheduler.py`
   - 实现 DAG 拓扑调度和线程池并行执行。

5. `llm.py`
   - 定义通用 LLM 接口。
   - 提供 `MockLLMClient` 便于无模型环境演示。

6. `pipeline.py`
   - 串联实体锚定、骨架生成、规则校验、调度扩展和最终聚合。

7. `cli.py`
   - 提供命令行演示入口。

## 四、最小可运行原型

输入：复杂查询文本。

输出：

- 结构化思维图；
- 规则处理日志；
- 并行扩展内容；
- 最终生成文本。

## 五、实验设计建议

1. 对比方法
   - 直接自回归生成。
   - 原生 SoT。
   - 本项目的 KG + GoT + Rule-guided SoT。

2. 数据集
   - HotpotQA：事实密集型多跳问答。
   - GSM8K：数学逻辑推理任务。
   - 自建长文本报告生成任务集。

3. 指标
   - 端到端延迟。
   - 事实准确率。
   - 幻觉率。
   - 逻辑一致性评分。
   - 结构完整度。
